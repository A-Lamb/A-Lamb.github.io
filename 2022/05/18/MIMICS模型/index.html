MIMICS模型

  • MIMICS模型，全称为密歇根微波冠层散射模型（Michigan microwave canopy scattering model），由ulaby等于1988年提出。

Ulaby F T , Mcdonald K , Sarabandi K , et al. Michigan Microwave Canopy Scattering Models (MIMICS)[C]// International Geoscience & Remote Sensing Symposium. IEEE, 1988.

  • 模型将森林植被在垂直方向上分为树冠层、树干层以及地表层。冠层包括树枝和叶， 在模型中分别建模为圆柱状介电体和扁平矩形，用联合概率密度分布函数表示其方位、直径、长度等等；树干层建模为介电圆柱体。

  • 模型中的微波散射由5个部分构成：
    ①植被冠层直接后向散射；
    ②植被层-下垫面地表和下垫面地表-植被层相互耦合作用的后向散射；
    ③下垫面地表直接后向散射；
    ④经过植被双程衰减的下垫面地表的直接后向散射；
    ⑤经过植被冠层衰减的树干层-地表和地表-树干层的二面角反射

MIMICS模型Fortran源码

Mimics1.5a

简介

https://codeocean.com/capsule/8976769/tree/v1
FROM registry.codeocean.com/codeocean/fortran:4.8.4-ubuntu14.04

  • Mimics1.5a is a code for simulating the microwave backscatter from a forest.
  • MIMICS1.5a was written at the University of Michigan, College of Engineering, Electrical Engineering and Computer Science Department, The Radiation Laboratory by Kyle McDonald, with some extensions by Leland Pierce.

运行环境

  • Fortran (4.8.4 )
  • Ubuntu 14.04

输入参数

首先按顺序编辑基本文件：

  1. configuration.input – general overall params for the simulation
    //配置文件：对各参数的总体描述和配置
  2. sensor.input – operating frequencies and incidence angles
    //传感器：包括频率和入射角
  3. environment.input – temperatures
    //环境：包括土壤温度、静水温度和植被温度（三者通常取同一值）
  4. ground.input – ground params and model
    //地表：包括静水表面参数（含盐量，单位ppt)、积雪表面参数（雪层厚度，单位m)、表面模型参数（表面散射模型的选择：几何光学模型 /物理光学模型 /小扰动模型 )
  5. dielectric_ground_table.input – optionally specify the ground dielectric constant
    //表面相对介电常数输入表: C波段、L波段、P波段
  6. dielectric_snow_table.input – optionally specify the snow layer dielectric constant
    //积雪表面相对介电常数

然后选择性编辑植被文件：
// 以下数据均存在两种输入方式：一种包括最小值、最大值和步长三个参数；另一种从表格中读取具体数据

  1. trunk_and_gross_canopy.input
    // 树干几何形状和总冠层特征：树干重量含水率、树干干密度、冠层密度（每平方米树的数量）、冠层厚度（m)、树干直径（cm）、树干长度（m)、树干直径标准差（cm)、树干长度标准差（m)；
    // 树干分布功能数据：树干方向、树干尺寸、功能描述
  2. dielectric_trunk_table.input
    // 树干相对介电常数表: C波段、L波段、P波段
  3. histogram_trunk_diam.input
    // 树干直径尺寸分布直方图：树干胸径（cm)
  4. trunk_hght_f.f
    // 利用树干直径计算树高的函数
  5. branch_primary.input
    // 与一级枝条有关的冠层几何参数：枝条重量含水率、枝条干密度、枝条长度、枝条直径、枝条密度（每平方米分枝数）
    // 一级枝条分布功能数据：枝条方向、功能描述
  6. dielectric_branch1_table.input
    // 一级枝条相对介电常数表: C波段、L波段、P波段
  7. branch_secondary.input
    // 与二级枝条有关的冠层几何参数：枝条重量含水率、枝条干密度、枝条长度、枝条直径、枝条密度（每平方米分枝数）
    // 二级枝条分布功能数据：枝条方向、功能描述
  8. dielectric_branch2_table.input
    // 二级枝条相对介电常数表: C波段、L波段、P波段

枝条包括4个以上等级，每个等级均包括冠层几何参数和相对介电常数两个文件

  1. leaf.input
    // 与叶片有关的冠层几何参数：叶片重量含水率（（湿重-干重）/湿重）、叶片干密度、叶片厚度、叶片直径、叶片密度（每平方米叶片数）
    // 叶片分布功能数据：均匀分布、planophile展型、erectophile直立叶株型（如玉米）、plagiophile、extremophile、uniform、spherical

  2. dielectric_leaf_table.input
    //叶片相对介电常数表

  3. needle.input
    // 与针叶有关的冠层几何参数：针叶重量含水率（（湿重-干重）/湿重）、针叶干密度、针叶长度、针叶直径、针叶密度（每平方米叶片数）
    // 针叶分布功能数据描述：均匀分布

  4. dielectric_needle_table.input
    // 针叶相对介电常数表

  5. parameter_nesting.input
    // 冠层参数嵌套顺序：列出了分配给每个参数的数字，较小的数字先循环
    // 共61个参数（包括积雪表面），详见表1。

  6. parameter_nesting.input
    // 不使用最小值-最大值-步长格式的输入参数
    // 可利用src文件夹中“make_table.f”文件创建
    //必须最后运行

输出文件

  1. forest_kappa_mats.out: Kappa matrices of individual layers
    // 每一层的Kappa矩阵
  2. forest_phase_crown.out: Phase differences for each crown-layer mechanism
    // 树冠各组分的相位差
  3. forest_phase_mats.out: Phase matrices for each full-canopy mechanism
    // 每一层的相位矩阵
  4. forest_phase_terms.out: Phase differences for each full-canopy mechanism
    // 整个冠层的相位差
  5. forest_phase_trunk.out: Phase differences for each trunk-layer mechanism
    // 树干层和地面层相位差
  6. forest_sigma_cross.out: Cross-polarized Sigma-0’s for each full-canopy mechanism
    //整个冠层交叉极化后向散射系数
  7. forest_sigma_like.out: Co-polarized Sigma-0’s for each full-canopy mechanism
    // 整个冠层同极化后向散射系数
  8. forest_trans_mats.out: Transmissivity matrices for individual layers
    // 每一层透射率矩阵
  9. forest_trans.out: Transmissivity values for each individual layer
    // 每一层透射率
  10. mimics2.data: Experimental
    //地表反射矩阵；树冠和树干M-矩阵
  11. polarimetric.out: Transformation matrices for each full-canopy mechanism
    //整个冠层反射率矩阵

注意：如果只需要得到不同极化方式下的后向散射系数，只需打开6、7文件。

在输出文件中，所有不变的数据被显示在顶部，变化的参数以表格形式输出。
